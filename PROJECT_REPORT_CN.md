# AE + GMM/VBGMM 手写数字生成项目课题报告

## 1. 课题概述

### 1.1 课题名称

基于 Autoencoder 与高斯混合模型的 MNIST 手写数字生成研究

### 1.2 课题目标

本项目以 MNIST 手写数字数据集为对象，探索一种结合非线性表示学习与经典概率生成模型的方法。项目的目标是：

- 对 28x28 的手写数字图像建立概率分布模型；
- 在低维潜变量空间中学习数字样本的多模态结构；
- 从学习到的分布中采样，生成新的手写数字图像；
- 比较 `EM-GMM` 与可自动裁剪成分的 `VBGMM` 两种混合建模方式；
- 通过似然、可视化和参数扫描来分析模型效果。

简言之，这个项目研究的生成建模链路是：

```
原始图像 -> Autoencoder Encoder -> 潜变量空间 -> VBGMM/GMM -> Decoder -> 生成图像
```

### 1.3 我完成的主要工作

从项目实现角度看，我完成的工作主要有以下几类：

1. 搭建了 MNIST 数据读取与预处理流程，统一把图像整理为 784 维归一化向量。
2. 实现了 `Autoencoder`，用于学习非线性潜变量表示。
3. 手工实现了 `GMM + EM`，支持 `full / diag / spherical` 三种协方差形式。
4. 手工实现了 `VBGMM`，通过变分贝叶斯推断实现混合成分的自动裁剪。
5. 设计了 `perclass` 和 `global` 两种训练模式，用于按类别生成和统一生成。
6. 完成了采样生成、原型图可视化、测试集打分、K 扫描和 t-SNE 可视化等评估工具。
7. 打通了 `AE + VBGMM` 的完整工程流程，实现了非线性表示学习 + 概率混合建模的生成系统。

## 2. 项目整体技术路线

从代码结构看，项目由以下几个核心模块组成：

- `data_loader.py`：负责加载 MNIST 数据，并统一转成 `[0, 1]` 范围的 `float32` 向量。
- `autoencoder.py`：实现 MLP Autoencoder，用于学习非线性潜变量表示。
- `gmm.py`：手写实现 GMM，使用 EM 算法训练。
- `vbgmm.py`：手写实现变分贝叶斯高斯混合模型，用于自动裁剪冗余成分。
- `representation.py`：管理 AE 表示层，负责编码、解码与 checkpoint 兼容。
- `train.py`：训练入口，支持 `perclass` / `global`、`gmm` / `vbgmm` 两个维度的组合。
- `generate.py`：从已训练模型中采样，并将潜变量通过 `AE decoder` 还原为图像。
- `evaluate.py`：用于 K 值扫描、测试集打分以及 t-SNE 可视化。

这一设计说明项目并不是简单调用现成库，而是围绕"表示学习 + 概率生成建模 + 实验评估"搭出了一条完整研究链路。

## 3. 数据与预处理方法

### 3.1 数据集

项目使用的是 MNIST 手写数字数据集，类别为 0 到 9，共 10 类。每张图像大小为 28x28，共 784 个像素。加载后，每张图像被展平成 784 维向量。

### 3.2 数据加载方式

`data_loader.py` 采用了三级回退策略：

1. 优先使用 `torchvision.datasets.MNIST` 自动下载与读取；
2. 若 `torchvision` 不可用，则从可用镜像下载原始 `idx` 格式压缩文件；
3. 若原始文件已存在，则直接解析二进制文件。

该模块最终输出：

- `X_train, y_train`
- `X_test, y_test`

其中：
- `X_*` 的形状为 `(N, 784)`；
- 像素值被归一化到 `[0, 1]`；
- 标签是整数类型。

### 3.3 为什么要做降维

MNIST 原始维度为 784，直接在这个空间上做 GMM 有两个明显问题：

- 协方差矩阵维度高，参数数量大，估计不稳定；
- EM 训练和采样成本较高，容易出现病态协方差或数值不稳定。

因此项目使用 Autoencoder 提取非线性潜变量表示，把高维图像压缩到低维连续表示空间，再在这个空间中做混合高斯建模。这是本项目最关键的方法设计之一。

## 4. 主要方法：Autoencoder 表示学习

### 4.1 Autoencoder 方法原理与实现

我实现了一个多层感知机结构的自编码器。其结构为：

- **Encoder**：`784 → 512 → 256 → latent_dim`
- **Decoder**：`latent_dim → 256 → 512 → 784`

其中：
- 编码器把原始图像压缩到低维潜变量空间；
- 解码器把潜变量重构回像素空间；
- 输出层使用 `Sigmoid`，保证像素范围位于 `[0, 1]`；
- 训练时使用重构损失（BCE）来学习一个适合图像表达的非线性潜空间。

这个模块的意义在于：它不再像 PCA 那样只能学习线性子空间，而是试图学习一个更贴近数字图像流形的非线性表示。

### 4.2 参数设置

训练脚本中支持的 AE 参数包括：

- `ae_latent_dim`：潜变量维度（推荐 16 ~ 64）
- `ae_epochs`：训练轮数（推荐 30 ~ 50）
- `ae_batch_size`：批大小（推荐 256）
- `ae_lr`：学习率（推荐 1e-3）
- `ae_ckpt`：加载已有自编码器参数

## 5. 主要方法：GMM 与 EM 算法

### 5.1 GMM 的基本思想

高斯混合模型假设数据由多个高斯分布加权混合得到。即一个样本 `x` 的概率密度写成：

```
p(x) = sum_k pi_k * N(x | mu_k, Sigma_k)
```

其中：
- `pi_k` 是第 `k` 个成分的混合权重；
- `mu_k` 是均值；
- `Sigma_k` 是协方差矩阵。

这个模型适合描述"同一类别内部存在多个书写风格"的情况。比如数字 2 有多种写法，单个高斯很难拟合，多高斯混合更合理。

### 5.2 为什么 GMM 适合这个课题

MNIST 中同一个数字内部具有明显的多模态特征：

- 数字 1 有直线型、带弯钩型等不同写法；
- 数字 7 有带横线和不带横线的不同风格；
- 数字 2、3、5 的弯曲方式也存在明显差异。

因此，把一个数字类别看成由多个高斯成分构成，是一个非常自然的建模思路。

### 5.3 EM 算法原理

由于 GMM 的隐变量是"样本属于哪个成分"，直接最大化似然较困难，所以项目用 EM 算法迭代求解。

#### E 步

计算责任度，也就是样本 `x_n` 属于第 `k` 个高斯成分的后验概率：

```
r_nk ∝ pi_k * N(x_n | mu_k, Sigma_k)
```

代码中先计算对数概率，再做 `log-sum-exp` 归一化，提高数值稳定性。

#### M 步

在责任度固定时，重新估计参数：

- `N_k = sum_n r_nk`
- `pi_k = N_k / N`
- `mu_k = sum_n r_nk x_n / N_k`
- `Sigma_k = sum_n r_nk (x_n - mu_k)(x_n - mu_k)^T / N_k`

项目根据不同协方差形式，分别实现了：
- `full`：完整协方差矩阵；
- `diag`：对角协方差；
- `spherical`：球形协方差。

### 5.4 本项目中的 GMM 实现细节

`gmm.py` 的实现有几个关键点：

#### 1. K-means++ 风格均值初始化

模型不是随机直接取 K 个点，而是先随机选一个点，再按"离现有中心越远，越容易被选中"的方式初始化均值。这比普通随机初始化更稳定，能缓解 EM 陷入差局部最优的问题。

#### 2. 全局方差初始化协方差

初始协方差不是完全随机，而是用数据整体方差初始化每个成分的协方差，有利于训练初期保持数值稳定。

#### 3. log-sum-exp 数值稳定技巧

在 E 步中，项目通过 `np.logaddexp.reduce` 计算归一化项，避免直接指数运算导致下溢。

#### 4. 协方差正则化

项目为协方差加入 `reg_covar`，默认 `1e-6`，避免矩阵奇异或方差为零。

### 5.5 协方差类型的含义

本项目支持三种协方差结构，含义如下：

- `full`：每个成分都有完整协方差矩阵，表达能力最强，但参数最多、计算最慢；
- `diag`：每个成分的各维度独立，只估计每维方差，效率较高；
- `spherical`：每个成分只有一个方差标量，假设最强，表达能力最弱。

README 中推荐 `diag` 作为默认选项，这是一个典型的工程折中：在 AE 潜空间中，对角协方差往往都能在表达能力和稳定性之间取得平衡。

## 6. 主要方法：VBGMM 变分贝叶斯高斯混合模型

### 6.1 为什么还要做 VBGMM

标准 GMM 需要事先指定混合成分数 `K`。但实际中往往不知道一个数字类别内部到底需要多少种"书写原型"。如果 `K` 过小，模型表达能力不足；如果 `K` 过大，又会引入冗余成分与过拟合风险。

为了解决这个问题，项目进一步实现了 VBGMM。

### 6.2 VBGMM 的核心思想

VBGMM 不再只求一个点估计，而是对参数的后验分布做近似推断。它给混合权重、均值和精度都加上先验分布，并通过变分推断更新这些后验参数。

本项目的先验设计为：

- `pi ~ Dirichlet(alpha_0)`
- `mu_kd ~ Normal(m_0d, 1 / (beta_0 * lambda_kd))`
- `lambda_kd ~ Gamma(a_0, b_0)`

这里 `lambda_kd` 是精度，即方差的倒数。

### 6.3 变分分布设置

项目在 `vbgmm.py` 中使用了均值场近似，令后验分解为：

- `q(pi) = Dirichlet(alpha)`
- `q(mu_kd) = Normal(m_kd, 1 / (beta_k * a_kd / b_kd))`
- `q(lambda_kd) = Gamma(a_kd, b_kd)`
- `q(Z) = Product Categorical(r_n)`

### 6.4 VBGMM 的更新思路

与 EM 类似，VBGMM 也在"责任度更新"和"参数更新"之间交替，但目标函数不再是对数似然，而是 ELBO，即证据下界。

项目中：
- E 步根据当前后验参数，计算每个样本对每个成分的责任度；
- M 步更新 `alpha_`、`beta_`、`m_`、`a_`、`b_`；
- 每次迭代后计算 ELBO，并用 ELBO 的增量判断收敛。

### 6.5 VBGMM 相比普通 GMM 的优势

这个实现最重要的优势是：
- 可以设置一个偏大的 `K`，再让模型自动裁剪不重要的成分；
- 通过 Dirichlet 先验实现稀疏化；
- 更适合做模型选择，减少人工反复试 K 的成本。

在代码中，是否"有效成分"由权重阈值 `weights > 1e-3` 判断，最后通过 `effective_n_components` 输出保留下来的成分数。

### 6.6 本项目中 VBGMM 的限制

该实现目前只支持对角协方差形式，不支持 `full` 和 `spherical`。这说明项目在设计上优先考虑了数值稳定性和实现复杂度。

## 7. 建模策略：按类建模与全局建模

`train.py` 支持两种训练模式：

### 7.1 perclass 模式

为每个数字类别分别训练一个模型，也就是：
- 数字 0 训练一个 GMM/VBGMM；
- 数字 1 训练一个 GMM/VBGMM；
- ...
- 数字 9 训练一个 GMM/VBGMM。

这种做法的优点是：
- 每个模型只拟合单一数字，任务更纯净；
- 生成时可以精确控制类别；
- 更容易分析每个数字内部的多模态结构。

这也是当前项目默认采用的模式。

### 7.2 global 模式

用一个统一模型拟合所有数字。其优点是结构更简洁，但难点在于：
- 类间差异很大；
- 模型既要区分类别，又要表达类内变化；
- 若没有显式标签约束，生成样本类别可能混杂。

从建模目标看，`global` 更像是"统一生成器"，而 `perclass` 更像是"类别条件式但无需神经网络"的生成方案。

## 8. 生成方法

### 8.1 生成流程

无论使用 GMM 还是 VBGMM，生成流程都遵循以下步骤：

1. 根据混合权重采样一个成分 `k`；
2. 从该成分对应的高斯分布中采样低维向量 `z`；
3. 调用 `AE decoder` 把潜变量还原到像素空间；
4. 将结果裁剪到 `[0, 1]` 并 reshape 成 `28x28` 图像。

### 8.2 原型图像可视化

`generate.py` 还支持 `--show_prototypes`，其做法是把各混合成分的均值向量直接逆变换回图像空间。这样可以观察：
- 每个成分学到了什么典型书写风格；
- 不同成分之间是否真的形成了风格分工；
- 模型是否存在重复成分或异常成分。

这在课题汇报中很有价值，因为它能直观展示"混合模型捕捉多模态结构"的证据。

### 8.3 锐化后处理

生成脚本中还加入了一个简单的图像后处理：
- 每张图先做对比度拉伸；
- 再做 gamma 变换。

这个步骤不改变模型本身，但有助于减轻经典概率模型生成图像常见的"发灰、发糊"问题。这是一个实用的工程增强，而不是核心建模方法。

## 9. 评估方法

本项目的评估方式比较完整，既包含定量指标，也包含结构性可视化。

### 9.1 测试集分数

`evaluate.py` 的基础指标是测试集 score per sample。对于普通 `GMM`，这个分数是平均 log-likelihood；对于 `VBGMM`，这个分数对应的是基于变分推断得到的 ELBO proxy。

这意味着：
- 在同一种模型内部，这个分数可以用于比较不同参数设置；
- 在 `GMM` 和 `VBGMM` 之间，不宜把数值直接做绝对横向比较；
- 更合理的做法是结合生成效果、有效成分数和可视化共同判断。

### 9.2 GMM 组件数 K 扫描

通过 `--k_sweep`，项目在单个数字上做：
- 80/20 训练验证划分；
- 训练不同 `K` 的 GMM；
- 比较验证集对数似然。

这一步是标准的模型选择实验，用于寻找固定 K 的较优设置。

注意：这里扫描的是普通 GMM，而不是 VBGMM，因为 VBGMM 的动机本来就是减少手动选 K 的需要。

### 9.3 t-SNE 可视化

通过 `--tsne`，项目把真实样本与生成样本一起投影到二维空间，看两者在低维嵌入中是否重叠。这一评估方法主要用于直观观察：
- 生成样本是否落在真实数据流形附近；
- 是否出现模式坍缩或偏移；
- 生成分布和真实分布的结构相似性如何。

## 12. 方法局限性分析

### 12.1 生成质量仍受表示层与高斯假设共同限制

虽然后端使用了 Autoencoder，但混合模型仍然建立在局部高斯假设上。因此生成图像依然可能出现：
- 边缘模糊；
- 笔画平均化；
- 细节不够锐利。

### 12.2 两阶段解耦训练的缺陷

AE 和 GMM/VBGMM 是分别训练的，没有联合优化目标函数。这导致：
- AE 的潜空间分布不一定适合 GMM 拟合；
- GMM 采样的点可能落在 Decoder 的"流形外"区域；
- 生成质量不如端到端训练的 VAE 或 GAN。

### 12.3 `diag` 协方差表达能力有限

当前最常用配置采用对角协方差，意味着默认不同潜变量维度之间条件独立。这会限制模型对复杂相关结构的表达能力。

### 12.4 VBGMM 实现仍有简化

当前 VBGMM 版本仅支持对角精度结构，且先验均值固定为 0，属于教学化、简化版实现，离完整的贝叶斯高斯混合模型还有一定距离。

如果后面你还要继续扩展这个课题，可以考虑以下方向：

### 14.2 提升生成质量

- 增加后处理，如自适应阈值、边缘增强；
- 把普通 Autoencoder 升级为 `VAE`，使潜空间更适合采样；
- 对每一类采用不同的最优 `K`，而不是统一固定 `K=10`。

### 14.3 联合优化

- 交替训练 AE 和 GMM，使 latent 分布更接近高斯混合；
- 引入重构 - 生成联合损失函数。

### 14.4 扩展实验对比
